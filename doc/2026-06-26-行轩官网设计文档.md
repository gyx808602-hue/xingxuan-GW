# 行轩官网设计文档

## 1. 项目目标

基于 React 技术栈为“行轩”创建一个简洁的科技类官网首页，整体风格采用“企业科技结构 + 轻未来感视觉”的融合方案，重点突出科技产品展示与技术能力表达。

## 2. 设计原则

- 结构简洁：只保留官网首页核心信息模块，避免堆砌内容。
- 科技感克制：使用深蓝渐变、描边卡片、轻微发光与数据化排版，不走过度炫技路线。
- 可扩展性优先：当前先实现首页，后续可方便扩展更多页面与接口数据。
- 转化路径清晰：访问者能快速理解“行轩是谁、能做什么、为何值得合作、如何联系”。

## 3. 目标用户

- 希望快速了解“行轩”品牌与技术能力的潜在客户
- 关注官网视觉表达与数字化能力展示的企业方
- 需要基础品牌官网的科技型团队或合作伙伴

## 4. 首页信息架构

### 4.1 Header 顶部导航

- 左侧展示品牌名“行轩”
- 右侧展示基础导航项：
  - 首页
  - 技术能力
  - 解决方案
  - 联系我们
- 移动端收敛为简洁堆叠或按钮式导航，不做复杂抽屉交互

### 4.2 Hero 首屏

- 主标题：强调技术能力与业务增长
- 副标题：一句话说明行轩的品牌定位
- CTA：
  - 了解能力
  - 查看方案
- 右侧辅以科技感数据卡片，强化“产品/能力展示”属性

### 4.3 Capabilities 技术能力区

- 使用 4 张能力卡片表达核心方向
- 计划展示能力：
  - 产品官网构建
  - 前端工程能力
  - 数字品牌表达
  - 方案快速落地

### 4.4 Solutions 解决方案区

- 使用 3 张方案卡片展示典型交付场景
- 计划展示方向：
  - 企业科技官网
  - 产品展示页面
  - 技术解决方案落地页

### 4.5 Brand Value 品牌价值区

- 用简洁数字和短句突出优势
- 让页面不只像“展示页”，而更像“可信赖的科技品牌官网”

### 4.6 Contact + Footer 联系区与页脚

- 联系引导文案
- 邮箱/电话/地址占位信息
- 页脚收束品牌信息与版权文本

## 5. 视觉策略

### 5.1 色彩

- 主背景：深蓝黑渐变
- 强调色：科技蓝、青蓝
- 辅助色：浅灰蓝、柔和白

### 5.2 排版

- 标题大字突出，强调科技感与品牌感
- 正文保持清晰、可读、不过度装饰
- 卡片区采用整齐网格布局，保持工程化秩序感

### 5.3 组件表现

- 卡片使用圆角、细描边、轻阴影
- 按钮使用渐变或高亮边框
- 背景加入轻微径向光晕，避免纯色背景过于平

## 6. 技术实现策略

- 使用 Vite 初始化 React 项目，降低搭建成本
- 采用函数组件拆分页面模块
- 使用单独 CSS 文件管理页面样式，保持实现简单直接
- 先完成纯静态首页，不引入状态管理、路由或复杂依赖

## 7. 文案风格

- 关键词聚焦：技术、产品、能力、方案、落地
- 语气专业、克制、干净
- 避免空泛口号与营销词过多堆叠

## 8. 本次交付范围

本次仅交付一个可运行的 React 官网首页，包括：

- React 项目初始化
- 首页模块拆分
- 科技感样式实现
- 基础响应式适配
- 本地构建验证

## 9. 非本次范围

- 后端接口接入
- CMS 内容管理
- 多页面路由系统
- 表单提交与邮件发送
- 多语言系统
